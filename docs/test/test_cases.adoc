== Test Cases

=== Test Case 01

* Bei anlegen eines neuen Benutzer muss das eingegebene Passwort im Backend bzw. im Frontend auf die Mindestlänge verglichen werden.
* Wenn es zu kurz ist, muss im Frontend eine passende Fehlermeldung ausgegeben werden und darf nicht als Passwort in der Datenbank gespeichert werden. Der Prozess muss wiederholbar sein.
* Wenn das Passwort lang genug ist, darf dieses angelegt werden.
=== Test Case 02

* Bei anlegen eines neuen Benutzer darf der Benutzername noch nicht vergeben sein Dies muss im Backend verglichen werden. 
* Wenn er bereits vergeben ist, muss im Frontend eine passende Fehlermeldung ausgegeben werden und darf nicht als Benutzername in der Datenbank gespeichert werden. Der Prozess muss wiederholbar sein.

* Wenn Benutzername noch nicht vergeben ist, darf dieser angelegt werden.

=== Test Case 03

* Bei anlegen eines neuen Benutzer darf der Benutzername nicht zu kurz sein. Dies muss im Backend verglichen werden. 
* Wenn es zu kurz ist, muss im Frontend eine passende Fehlermeldung ausgegeben werden und darf nicht als Benutzername in der Datenbank gespeichert werden. Der Prozess muss wiederholbar sein.
* Wenn Benutzername  nicht zu kurz ist, darf dieser angelegt werden.
=== Test Case 04

* Beim Anmelden eines Benutzers darf kein falsches Passwort zu dem Benutzernamen eingegeben werden
Dies muss im Backend verglichen werden. 
* Wenn die beiden Daten nicht zusammenpassen, muss eine entsprechenden Fehlermeldung ausgeben werden. Der Benutzer darf dann nicht eingeloggt werden. Der Prozess muss wiederholbar sein.
* Bei übereinstimmender Daten, darf der Benutzer angemeldet werden.

=== Test Case 05

* Beim Button "Abrechen und Zurück" dürfen bereits eingegebene Daten nicht an das Backend übermittelt werden 

=== Test Case 06
//wo wird das überprüft? 
* Bei Freitextfeldern darf keine SQL-Injection möglich sein.
* Bestimmte Zeichen dürfen in diesen Feldern nicht benutzt werden. Dies muss vor der Eingabe des Textes in ein SQL-Statement überprüft werden. Wenn es verbotene Zeichen besitzt dürfen diese Daten nicht übertragen werden  

=== Test Case 07

* Beim anlegen eines neuen Benutzer muss jedes Feld ausgefüllt sein. Das Überprüfen muss nach dem drücken des Knopf "Benutzer anlegen" stattfinden. 
* Wenn dies nicht der Fall ist, muss eine Fehlermeldung ausgeben werden und die unvollständigen Daten dürfen nicht an das Backend übermittelt werden. Eventuell werden auch ausgefüllte Felder durch andere Test Cases abgelehnt.
* Wenn die alle Felder ausgefüllt sind, wird kein Fehler ausgeben. Jedoch kann durch einen anderen Testfall ein Fehler ausgeben werden.
