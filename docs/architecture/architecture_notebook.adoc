== Architekturziele und Philosophie

Das System ist eine Webanwendung für das Verwalten und Planen von Mitarbeitern und Projekten. Die Webanwendung kann bloß aus dem Intranet benutzt werden und erfordert ein Login. Es wird maximal von 50 Benutern gleichzeitig genutzt, jedoch ist dies realistisch gesehen nicht zu erwarten. In der Realität werden es vermutlich nicht mehr als 7 Benutzer gleichzeitig sein. Der Zugriff erfolgt über Computer über die keine Spezifiktationen bekannt sind.

Da mit dem System geplant werden soll, ist die Datenkorrektheit und -integretät wichtig, sowie eine hohe Verfügbarkeit. Die Auslieferung bzw. die Benutzung erfolgt über GitHub und Docker. Es gibt kein Altsystem mit dem das neue Synchrönisiert werden soll. Personenbezogene Daten müssen im einklang mit der DSGVO behandelt werden, ebenso soll die Webanwendung grundlegende Sicherheitskonzepte erfüllen. 

Als Vorraussetzung ist eine schnelle und simple Navigation bzw. Anlegung von Projekten bzw. Benutzern erforderlich. 
Da in der gegebenen Zeit nicht klar ist, ob das System in den geforderten Umfang realisierbar ist, soll es erweiterbar sein. Außerdem soll es rudimentär Wartbar sein
