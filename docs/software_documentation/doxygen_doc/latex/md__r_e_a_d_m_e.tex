\hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysubsection{Unified Process -\/ Templates}\label{md__r_e_a_d_m_e_autotoc_md2}
Dieses Repositority enthält Vorlagen im Ascii\+Doc-\/\+Format für die Arbeit mit dem Open Unified Process. Die Vorlagen basieren auf den Templates der offiziellen Open\+UP Dokumentation in der Version 1.\+5x aus dem Eclipse Process Framework (EPF) und wurden bis auf geringe inhaltliche / strukturelle Anpassungen lediglich auf deutsch übersetzt.\hypertarget{md__r_e_a_d_m_e_autotoc_md3}{}\doxysubsubsection{Hinweise}\label{md__r_e_a_d_m_e_autotoc_md3}

\begin{DoxyItemize}
\item Im Unified Process gibt es eine \char`\"{}\+Supplementary Specification\char`\"{}, die Anforderungen enthält, die nicht als Use Case Model spezifiziert werden können. Das sind z.\+B. nicht-\/funktionale und systemweite Anforderungen. Im Open\+UP (und in diesem Repository) heißt dieses Dokument \char`\"{}\+System-\/wide Requirements\char`\"{}.
\item Falls Sie Fehler korrigieren oder Verbesserungen machen möchten, können Sie dies gern über einen {\itshape Pull Request} tun.
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md4}{}\doxysubsubsection{Referenzen}\label{md__r_e_a_d_m_e_autotoc_md4}

Die Datei {\itshape default-\/attributes.\+inc.\+adoc} im {\bfseries{docs/\+\_\+includes}} Verzeichnis enthält die gemeinsamen Dokumentenattribute für alle Ascii\+Doc-\/\+Dokumente. In jedem Dokument selbst können, nach dem include der Datei, entsprechend noch extra benötigte Attribute mit aufgenommen werden.\hypertarget{md__r_e_a_d_m_e_autotoc_md6}{}\doxysubsubsection{Projektnamen definieren}\label{md__r_e_a_d_m_e_autotoc_md6}
In der {\itshape default-\/attributes.\+inc.\+adoc} Datei ist am Anfang der {\bfseries{Projektname}} definiert, welcher in jedem Dokument im Title verwendet wird. Ersetzen Sie hier bitte {\ttfamily \texorpdfstring{$<$}{<}Projektname (T00)\texorpdfstring{$>$}{>}} mit Ihrem Belegthema\+:

Im Verzeichnis {\bfseries{belegabgabe\+\_\+se1}} finden Sie die Vorlagedatei {\itshape se1\+\_\+belegabgabe\+\_\+t00.\+adoc}, welche alle Ihre erzeugten Dokumente für die Abgabe als PDF in \uline{ein} Dokument bündelt.

(Nutzen Sie nicht die Projektvorlage {\bfseries{Projektstruktur\+\_\+\+Open\+UP-\/\+Templates}}, kopieren sie sich die Vorlagedatei {\itshape se1\+\_\+belegabgabe\+\_\+t00.\+adoc} in Ihr Projektrepository)

Folgende Schritte sind für eine Belegabgabe durchzuführen\+:


\begin{DoxyEnumerate}
\item Ändern Sie die Themennummer {\bfseries{t00}} in der Vorlagedatei {\itshape se1\+\_\+belegabgabe\+\_\+t00.\+adoc} in Ihre Themennummer (t01, t02, ...).
\item Inhalt der Vorlagedatei anpassen\+:
\begin{DoxyItemize}
\item Ist in Ihrem Projekt in der Datei {\itshape docs/\+\_\+includes/default-\/attributes.\+inc.\+adoc} der Projektname im Attribut {\ttfamily \+:project-\/name\+:} nicht gesetzt bzw. nutzen Sie eine andere Struktur, können Sie im Dokumententitel nach dem {\ttfamily \+:} das {\ttfamily \{project-\/name\}} mit Ihrem Projektthema ersetzen\+:
